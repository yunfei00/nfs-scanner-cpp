# 扫描状态机说明

## v0.3.0 功能范围

v0.3.0 只实现扫描流程控制：读取扫描区域配置、生成二维扫描路径、模拟扫描推进、暂停/继续/停止、日志输出和状态栏进度显示。

本阶段不接入真实频谱仪，不保存数据，不生成热力图，不调用真实运动控制。

## ScanConfig 字段

- `startX/startY/startZ`：扫描起点坐标。
- `endX/endY/endZ`：扫描终点坐标。
- `stepX/stepY/stepZ`：扫描步长，当前二维路径使用 X/Y 步长，Z 固定为 `startZ`。
- `feed`：预留运动速度，当前状态机不发送运动命令。
- `dwellMs`：每个扫描点的模拟驻留时间，最小强制为 50 ms。
- `snakeMode`：是否启用蛇形扫描。
- `projectName/testName/outputDir`：测试描述信息，当前不持久化。

## 路径生成规则

`ScanPathPlanner` 根据起点和终点自动判断 X/Y 方向。`stepX` 和 `stepY` 始终为正数，方向由 start/end 决定。

终点会被纳入路径；如果步长不能整除扫描范围，最后一个点会使用终点坐标，避免明显越界。

## 蛇形扫描

启用蛇形扫描时，第一行按 X 正向扫描，第二行按 X 反向扫描，后续行依次交替，减少实际运动回程。

关闭蛇形扫描时，每一行都按相同 X 方向生成。

## ScanManager 状态流转

- `Idle`：就绪
- `Preparing`：准备中，生成扫描路径
- `Running`：扫描中，QTimer 推进扫描点
- `Paused`：已暂停，保留当前进度
- `Stopping`：正在停止
- `Stopped`：已停止
- `Finished`：已完成
- `Error`：错误

## 开始/暂停/继续/停止

- 开始：从 UI 读取配置，生成路径，清零进度并启动定时器。
- 暂停：仅在扫描中有效，停止定时器，按钮文字变为“继续”。
- 继续：仅在已暂停有效，从当前点继续推进。
- 停止：扫描中或已暂停时有效，停止定时器并保留当前进度。

## v0.4.0 数据存储计划

下一阶段计划引入扫描结果数据模型和本地数据保存流程，并为后续频谱仪采样、热力图和报告导出预留稳定的数据格式。
